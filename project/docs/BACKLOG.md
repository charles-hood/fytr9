# Post-v1 backlog

Ideas explicitly excluded from v1 (plan §2, §18). Record new feature ideas
here — never build them during v1. None of these should influence the v1
architecture beyond avoiding obvious dead ends.

- Score multiplier / combo mode.
- Boss encounters.
- Multiplayer: two-player alternating mode, co-op.
- Online leaderboards, accounts, telemetry, cloud saves.
- Full keyboard/gamepad rebinding UI, with persisted custom bindings and
  binding-conflict handling (v1.1 target; adds save-schema fields for
  bindings when it ships — §10.2, §10.8).
- Tracked tutorial-completion flag (once there is an interactive tutorial to
  track, beyond the static how-to-play screen).
- UI scaling beyond the base layout; dedicated colorblind palette presets.
- Alternate planets / palettes; campaign mode.
- Power-up drops, weapon upgrades, shields, RPG progression.
- CRT shader (unless trivially cheap and disabled by default).
- Mobile touch controls.
- An original composed soundtrack beyond the single optional loop (§9).
- Full localization (strings are externalized in v1; translation is not).
- Achievements, storefront SDKs, paid services, native extensions.
